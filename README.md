📌 Project Title
E-Commerce Database Management System using SQL

📖 Project Overview
This project is a Mini SQL Database Project based on an E-Commerce Management System. The main objective of this project is to design and manage a relational database for handling customers, products, orders, payments, and product reviews.


The project demonstrates practical SQL skills such as:
Database creation

1.Table relationships

2.Primary & Foreign Keys

3.Constraints

4.Data insertion

5.Data management

6.Query writing

7.Real-world database design

🛠️ Technologies Used
SQL
MySQL
MySQL Workbench

🗂️ Database Structure

The database contains the following tables:

1. Customers
   
Stores customer details such as:

Customer ID

Name

Email

Phone Number

Account Creation Date


2. Products

Stores product-related information:

Product ID

Product Name

Category

Price

Stock Quantity

Added Date


3. Orders
   
Contains order transaction details:

Order ID

Customer ID

Order Date

Order Status

Total Amount


4 Order Items

Maintains product-level order details:

Order Item ID

Order ID

Product ID

Quantity

Item Price


5. Payments

Stores payment information:

Payment ID

Order ID

Payment Date

Amount Paid

Payment Method


6. Product Reviews
   
Contains customer feedback and ratings:

Review ID

Product ID

Customer ID

Rating

Review Text

Review Date


🔗 Relationships Used

This project uses relational database concepts:

1.One customer can place multiple orders.

2.One order can contain multiple products.

3.One product can receive multiple reviews.

4.Payments are linked with orders.

5.Foreign keys are used to maintain data integrity between tables
